## Post 1

1/9

The core claim behind Styx is simple:

Software does not change human drives.
It amplifies, dampens, reroutes, or blinds feedback loops.

Most products fail because they mis-specify feedback, not because they lack features.

## Post 2

2/9

What moral language calls vice is often better modeled as a recurrent drive inside a control system:
greed, pride, envy, lust, wrath, gluttony, sloth.

The question is not how to erase them.
The question is how a system constrains them.

## Post 3

3/9

Humans do not primarily learn from instruction.
They learn from consequence density.

If the cost of indulgence is delayed, hidden, or externalized, behavior drifts.
If the signal is immediate, legible, and real, behavior calibrates.

## Post 4

4/9

This is the Human Vice Control System thesis:

drives compete,
excess creates pressure,
pressure produces correction,
stability is local and temporary.

Systems fail when the feedback path is severed, not when the drive exists.

## Post 5

5/9

Capitalism breaks when greed is detached from loss and competition.
Social media breaks when pride and wrath are rewarded but reputational decay disappears.
Consumer systems break when gluttony no longer meets visible bodily cost.

Same pattern. Broken feedback.

## Post 6

6/9

The right product question is:

"What feedback does the user receive, how fast, how truthful, and at what cost?"

If feedback is slow, fake, symbolic, or consequence-free, users will optimize the system instead of the problem.

## Post 7

7/9

Software is always acting as:

- a sensor
- a controller
- a delay line
- a gain amplifier
- a memory system

Design errors happen when software replaces reality instead of reporting it.

## Post 8

8/9

Behavioral physics gives a few hard rules:

Every reward must carry exposure to loss.
Every metric must map to real cost or benefit.
Every reputation system must have memory.
Every comparison must imply a plausible path.

These are stability conditions.

## Post 9

9/9

Styx exists because many behavioral failures are failures of consequence density.

The system tries to collapse delay, harden signal, preserve memory, and make loss possible again.

Software that respects feedback becomes stabilizing and durable.
